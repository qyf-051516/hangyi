# 航翼排班 — UI 美化 & 甘特图可视化升级

## 概览

在国创赛前（DDL: 2026-05-17）对系统进行视觉和演示效果升级，提升评委印象分。聚焦两个最高可见度页面：登录页和甘特图。

## 1. 登录页改造

### 目标
让评委打开系统第一眼就感受到专业度和科技感。

### 改动
- **动态航线背景**：纯 CSS/SVG 实现的发光线动画，模拟飞机航迹，不引入额外依赖
- **入场动画**：卡片加载时淡入 + 上浮效果
- **登录成功后过渡**：增加加载波纹动画，平稳过渡到仪表盘

### 涉及文件
- `web/src/views/login/Login.vue` — 唯一改动文件

### 非目标
- 不改动后端
- 不改动认证流程
- 不引入额外 npm 包

## 2. 甘特图升级

### 目标
将当前基础 div 甘特图升级为交互式可视化工具，展示排班系统的技术实力。

### 2.1 周视图 + 滚动时间轴
- 当前：仅支持单日查询
- 改为：按周展示，时间轴横向可滚动
- 左侧：人员姓名固定列
- 顶部：日期行（显示月/日/星期）
- 新增后端接口：`GET /api/schedules/gantt-range?startDate=xxx&endDate=xxx&groupId=xxx`，返回日期范围内所有排班数据

### 2.2 交互增强
- **工具提示**：悬停显示班次名称、起止时间、人员岗位/资质
- **班次颜色映射**：与班次模板配置的颜色一致，休息日显示浅灰标记
- **人员分组**：按班组分组显示，组间有分隔线

### 2.3 排班指标面板
- 在 Gantt 图表顶部新增指标卡片
- 展示内容：
  - 排班周期总览（起止日期、天数）
  - 各时段在岗人数统计（早班/中班/晚班）
  - 人员排班天数分布
- 数据来源：前端从前端对 gantt-range 返回数据做聚合计算（避免额外后端开发）

### 2.4 导出增强
- 截图中包含指标面板信息

### 涉及文件

#### 前端
| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `web/src/views/schedule/ScheduleGantt.vue` | 重写 | 核心改动，约 400 行 |
| `web/src/api/schedule.js` | 新增 | 添加 `getGanttRange` API 调用 |

#### 后端
| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `hangyi-schedule/.../ScheduleController.java` | 新增接口 | `ganttRange` 端点，复用现有 gantt 数据组装逻辑 |
| `hangyi-schedule/.../ScheduleService.java` | 新增方法 | 按日期范围批量查询排班 + 附加信息 |

### 非目标
- 不做拖拽调整班次（时间不够）
- 不做多视图切换（月视图）
- 不改动现有排班 CRUD 逻辑

## 3. 实现顺序

1. 后端：新增 gantt-range 接口
2. 前端：重写 ScheduleGantt.vue
3. 前端：改造登录页
4. 联调测试

## 4. 不涉及的部分
- 人员管理、班组管理、资质管理、班次模板等其他页面不做改动
- 移动端（uni-app）不做改动
- 后端排班引擎（Timefold）不做改动
- 数据库 schema 不做改动
